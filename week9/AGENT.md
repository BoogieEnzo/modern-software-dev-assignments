# Week9 Agent 执行任务线

## 任务总览

本项目目标：从"实时抓取"升级为"每日快照驱动"的稳定版本。

---

## 环境搭建

- [ ] **T001** - 修复 Python 环境：确认使用 Python 3.10+ 运行，确保 `make test` 能通过
- [ ] **T002** - 配置 SQLite 数据库连接，验证 `data/app.db` 可读写

---

## 核心功能：快照持久化

### 数据库层
- [ ] **T003** - 创建 `RepoSnapshot` SQLAlchemy 模型
  - 字段：`id`, `date`, `repos_json`, `generated_at`, `status`, `error_msg`
  - 参考 `plan.md` 的 schema 设计

### 服务层
- [ ] **T004** - 实现 `save_daily_snapshot(date, repos)` 函数
  - 写入快照到数据库
  - 处理重复日期覆盖

- [ ] **T005** - 实现 `load_today_snapshot(date)` 函数
  - 读取指定日期快照
  - 返回 None 如果不存在

- [ ] **T006** - 修改 `GET /api/trending/today` 接口逻辑
  - 优先从快照读取
  - 无快照时返回 503 + `"暂时无法更新"`

### API 层
- [ ] **T007** - 新增 `POST /api/internal/snapshot/run` 内部接口
  - 调用抓取逻辑
  - 成功后写入快照
  - 失败时记录错误日志
